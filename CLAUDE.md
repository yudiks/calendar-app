# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

This is a greenfield calendar application. Update this file once the project is scaffolded with commands, architecture, and structure.

## Working Rules

1. **Read the codebase first** — before starting any task, read the relevant code and write a plan in `tasks/todo.md`.
2. **Plan as a checklist** — the plan must be a markdown checklist of discrete todo items.
3. **Mark off as you go** — check off each item (`- [x]`) immediately after completing it, not in batch at the end.
4. **Explain each step** — after completing each todo item, write a short high-level explanation of what changed and why.
5. **Keep changes minimal** — make the smallest change that accomplishes the goal; avoid large rewrites or unrelated refactoring.
6. **Add a review section** — when all items are done, append a `## Review` section to `tasks/todo.md` summarizing the overall changes made.
